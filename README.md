# Caesar Zipher

Программа для динамической печати кодов маркировки **«Честный ЗНАК»** на принтерах VideoJet по протоколу Zipher.

## Использование

1. Подготовьте файл с кодами маркировки (один КМ на строку).
2. Запустите программу → нажмите **«Загрузить коды»** и выберите файл.
3. Нажмите кнопку **«#»** и проверьте настройки.
4. Нажмите **«Не подключен»** и дождитесь соединения с принтером.
5. Нажмите **«Остановлен»**, чтобы запустить печать.

## Требования к макету этикетки (CLARiSOFT)

Макет готовится в **CLARiSOFT**. Обязательные поля:

1. **Текстовое поле GTIN**  
   - Может быть фиксированным или переменным.  
   - Осмысленное имя (указывается на вкладке `General`).

2. **Текстовое поле серийного номера**  
   - Обязательно **переменное**.  
   - Осмысленное имя.  
   - **Фиксированная длина обязательна**: установите длину поля точно равной длине серийного номера в КМ (обычно 13 символов). Если поле короче - динамическая замена не сработает.

3. **Текстовые поля кодов проверки** (минимум одно, 91–99)  
   - Обязательно **переменные**.  
   - Осмысленные имена.  
   - **Фиксированная длина обязательна**: установите длину поля точно равной длине кода проверки. Если поле короче - динамическая замена не сработает.

4. **Штрихкод DataMatrix**  
   - Создаётся через **Composite Data Builder**.  
   - Должен содержать:  
     - `01` → ссылка на поле GTIN  
     - `21` → ссылка на поле серийного номера  
     - Минимум один `9x` → ссылка на поле кода проверки

### Обязательная правка XML

**CLARiSOFT** автоматически ставит разделитель **FNC1** (`~1`) во всех местах, где должен быть **GS** (`~d029`).  
FNC1 разрешён **только в самом начале** кода маркировки. Во всех остальных местах его нужно заменить на `~d029`.

Откройте файл макета в VS Code и замените в поле штрихкода:

Было:  
``` xml
...
<Field Name="DM">
    <FldType>Barcode</FldType>
    <CLSID>{7A4AA4CF-F5CD-11D4-8DAE-0050DAFE8A9F}</CLSID>
    ...
    <CalcData><![CDATA[~1010460107151395421SerialNumber~193CryptoPart]]></CalcData>
    ...
</Field>
...
```

Стало:
``` xml
...
<Field Name="DM">
    <FldType>Barcode</FldType>
    <CLSID>{7A4AA4CF-F5CD-11D4-8DAE-0050DAFE8A9F}</CLSID>
    ...
    <CalcData><![CDATA[~1010460107151395421SerialNumber~d02993CryptoPart]]></CalcData>
    ...
</Field>
...
```  

А также в составных полях внутри поля штрихкода:  

Было:
``` xml
...
<Object Index="5" Reference="">
    <DataType>0</DataType>
    <Default><![CDATA[~1]]></Default>
</Object>
...
```  

Стало:
``` xml
...
<Object Index="5" Reference="">
    <DataType>0</DataType>
    <Default><![CDATA[~d029]]></Default>
</Object>
...
```  

Подробная инструкция: https://support.bit-erp.ru/article/ru/93-sozdanie-shablona-printera-videojet.html

## Требования к файлу с кодами
1. Формат: .txt или .csv
2. Один код маркировки на строку
3. Код полный
4. GS должен быть экранирован как \u001d
