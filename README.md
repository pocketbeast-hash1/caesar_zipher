# Caesar Zipher
Программа для работы с принтером VideoJet по протоколу Zipher. Используется для динамической печати кода маркировки системы «Честный ЗНАК» на этикетке.

## Использование
1. Подготовить файл с КМ. Один КМ на строчку.
2. Запустить программу и нажать «Загрузить коды», а затем выбрать файл с КМ.
3. Нажать кнопку «#» и убедиться в правильности настроек.
4. Нажать кнопку «Не подключен» и дождаться подключения к принтеру.
5. Нажать кнопку «Остановлен», чтобы запустить работу программы.

## Требования к макету этикетки
Макет подготавливается в программе CLARISOFT.
1. На макете должно присутствовать текстовое поле с GTIN продукта.  
    1.1. Поле может быть фиксированным или переменным.  
    1.2. У данного поля должно быть осмысленное название, которое затем будет указано в настройках программы Caesar Zipher.  
    1.3. Поле можно вынести за границы этикетки, чтобы не печаталось.  
2. На макете должно присутствовать текстовое поле для серийного номера КМ.  
    2.1. Поле должно быть переменным, так как оно будет динамически изменяться посредством программы Caesar Zipher.  
    2.2. У данного поля должно быть осмысленное название, которое затем нужно указать в настройках программы Caesar Zipher.  
    2.3. Поле можно вынести за границы этикетки, чтобы не печаталось.  
3. На макете должно присутствовать текстовое поле как минимум для одного кода проверки (это те, которые в коде маркировки разделены символом \u001d).  
    3.1. Поля должны быть переменными, так как они будут динамически изменяться посредством программы Caesar Zipher.  
    3.2. У данных полей должно быть осмысленное название, которое затем нужно указать в настройках программы Caesar Zipher.  
    3.3. Поля можно вынести за границы этикетки, чтобы не печатались.  
4. На макете должен присутствовать штрихкод типа Datamatrix.  
    4.1. Штрихкод должен быть составлен с помощью Composite Data Builder.  
    4.2. В штрихкоде должен быть идентификатор 01 с указанием на поле с GTIN.  
    4.3. В штрихкоде должен быть идентификатор 21 с указанием на поле с серийным номером.  
    4.4. В штрихкоде должен быть как минимум один идентификатор из 91–99 с указанием на поле с кодом проверки. Чтобы выбрать нужный идентификатор из диапазона, нужно ориентироваться на шаблон кода маркировки, который вы заказываете.  

## Требования к файлу с кодами маркировки
1. Файл должен быть в формате .txt или .csv.
2. В файле должен содержаться один код на одну строчку.
3. Код маркировки должен быть полным.
4. Символ-разделитель GS должен быть экранирован и заменён на "\u001d".
